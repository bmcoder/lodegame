# Lodegame

Lodegame - веб-игра на Next.js, React, Phaser и Socket.io в духе Lode Runner, но с единым общим онлайн-миром. Игроки бегают по большой общей карте, собирают золото, строят стены и лестницы, прокачивают героя, используют экипировку, торгуют вещами и общаются в общем чате.

## Стек

- Next.js App Router, React, TypeScript
- Phaser 3 для игрового canvas, физики, врагов, стрельбы и карты
- Socket.io для общего мира, онлайн-игроков, чата, рынка и синхронизации ресурсов
- Zustand для локального состояния профиля, инвентаря, авторизации и кастомизации
- Tailwind CSS для интерфейса
- localStorage для аккаунтов, профиля, скина, прогресса и настроек

## Быстрый Запуск

Установить зависимости:

```bash
npm install
```

Запустить Next.js:

```bash
npm run dev
```

В отдельном терминале запустить сетевой сервер:

```bash
npm run dev:net
```

Открыть игру:

```text
http://localhost:3000
```

Socket.io сервер по умолчанию работает на:

```text
http://localhost:3001
```

Без авторизации войти в игровой мир нельзя. Прямой переход на `/game/1` вернет игрока в меню.

## Скрипты

```bash
npm run dev              # Next.js dev server
```

## Игровой Мир

Сейчас игра использует один общий мир. Видимая область canvas остается ограниченной, но карта значительно шире и выше, камера плавно следует за игроком.

Карта хранится на сервере в `server/world-state.json`. Это persistent world: построенные стены, лестницы и часть мирового состояния сохраняются между перезапусками сервера. Если файла нет или формат не подходит, сервер создает базовую карту.

Коды тайлов:

```text
.  пустота
#  камень/граница
S  земля
P  платформа/кирпич
L  лестница
D  дверь
X  выход/служебный тайл
```

Игроки могут:

- строить стену за 1 камень;
- строить лестницу за 1 железо;
- строить дверь за 1 камень и 1 железо;
- убирать стены, лестницы и двери;
- сохранять личную точку старта кнопкой `Моя база`;
- после потери жизни появляться на сохраненной точке старта.

Двери преграждают путь врагам, но игроки проходят через них свободно. Это позволяет строить базы с проходом для игроков и защитой от врагов.

Основная карта внизу игры по умолчанию скрыта и открывается кнопкой `КАРТА`. В модальном окне карты видны игрок, другие игроки, враги, золото, камень, железо и инкубаторы.

## Управление

```text
WASD / стрелки  движение
Пробел          прыжок
E / клик        выстрел
Q               испарить/выкопать блок рядом
R               супероружие
1               режим строительства стены
2               режим строительства лестницы
3               режим демонтажа
4               режим строительства двери
5               выключить строительство
```

Для мобильных есть touch-контролы.

Важно для чата: когда фокус находится в поле ввода чата, Phaser отключает игровые клавиши, поэтому WASD, русские буквы и пробел вводятся как текст.

Пауза останавливает локальную Phaser-сцену: игровое время, износ экипировки и действия героя не начисляются. Для других игроков герой на паузе исчезает из общего мира и онлайн-снимка, а после продолжения заново появляется на текущей позиции.

## Авторизация И Профиль

Авторизация локальная, через `localStorage`, без серверной базы пользователей. Вход, регистрация и выход находятся на `/settings`. В `useGameState.ts` хранится список локальных аккаунтов, текущий email, профиль, инвентарь, экипировка и кастомизация.

Nickname задается один раз при регистрации и дальше не редактируется.

Фиксированный хедер показывает:

- nickname или статус гостя;
- опытность;
- золото;
- камень;
- железо;
- жизни;
- кнопку настроек;
- кнопку входа/регистрации или выхода.

Подписи к компактным ресурсам выводятся через tooltip при наведении.

Профиль содержит:

- nickname;
- email;
- очки;
- золото;
- камень и железо для строительства;
- прокачку героя;
- инвентарь;
- надетую экипировку;
- личную точку старта.

## Кастомизация Героя

Редактор находится на `/customization`. Из настроек он открывается кнопкой `Редактор героя`.

Можно менять:

- кофту;
- штаны;
- головной убор;
- волосы;
- лицо;
- стиль;
- pixel-skin 16x16.

Pixel-skin сохраняется в `localStorage` и применяется в Phaser-текстуре героя. Если pixel-skin нарисован, он используется вместо стандартного блочного человечка.

В настройках `/settings` показывается сводка героя: аватар, золото, камень, железо, очки, надетые вещи, прокачка и содержимое рюкзака.

## Враги

Враги создаются в `MainScene.ts` и частично описываются в `levels.ts`.

Текущие типы:

- hauntedTrain - страшный паровозик;
- siren - сиреноголовый, стреляет звуковой волной;
- skibidiToilet - скибиди-унитаз, медленно ходит и кидает какашку в героя;
- walker/hunter/shooter - прочие враги, визуально стилизованы под монстров.

Инкубаторы постепенно выращивают новых врагов. После гибели врага инкубатор запускает новый цикл.

Выстрел игрока может нейтрализовать звуковую волну сиреноголового.
Выстрел игрока также может сбить какашку скибиди-унитаза.

## Золото, Камень, Железо И Суперпризы

Сервер случайно спавнит золото, камень и железо в общем мире. Золото остается валютой для покупок, ремонта, апгрейда и рынка. Камень и железо являются отдельными строительными ресурсами: стены строятся за 1 камень, лестницы строятся за 1 железо, двери строятся за 1 камень и 1 железо.

Материалы можно:

- собрать на карте;
- купить в магазине за золото;
- продать обратно в магазине.

Текущие цены магазина материалов:

- камень: купить за 1 золото, продать за 1 золото;
- железо: купить за 3 золота, продать за 2 золота.

Суперпризы появляются временно. Если игрок успевает подобрать предмет и в рюкзаке хватает места, он попадает в рюкзак. Если места нет, игра открывает окно героя/рюкзака и предмет остается на карте.

Вещи бывают разных типов:

- оружие;
- броня;
- сапоги;
- рюкзак;
- модуль.

Вещи имеют редкость, силу, вес, стоимость и срок службы. Срок службы каждого предмета - 1 час игрового времени.

## Рюкзак, Экипировка И Износ

Рюкзак находится в разделе `Герой`, а не в магазине. Вещи, которые надеты на героя, не занимают место в рюкзаке.

Предметы в рюкзаке не изнашиваются. Износ идет только у надетых вещей, то есть у предметов в `profile.equipment`.

Если срок службы надетой вещи закончился, она снимается/исчезает из экипировки.

Игрок может:

- надеть вещь;
- снять вещь;
- ремонтировать вещь;
- апгрейдить вещь;
- быстро продать вещь;
- выставить вещь на рынок.

Стоимость вещи зависит от оставшегося ресурса ношения. Почти сломанная вещь стоит меньше. Ремонт восстанавливает ресурс до 1 часа, апгрейд повышает силу и ценность вещи, но срок службы остается 1 час.

Большой рюкзак и общий вес груза уменьшают скорость бега и высоту прыжка. Прокачка силы снижает штраф.

Если игрок пытается подобрать суперприз, но вместимости рюкзака не хватает, предмет не забирается и не исчезает. Игра показывает сообщение с нужным и свободным весом, затем открывает окно героя/рюкзака, где можно надеть вещь, быстро продать вещь или выставить ее на рынок.

## Прокачка

За золото можно улучшать:

- скорость бега;
- скорострельность;
- урон выстрела;
- броню;
- высоту прыжка;
- размер рюкзака;
- силу.

Цены задаются в `upgradePrices` в `src/app/hooks/useGameState.ts`.

## Рынок Игроков

Магазин открывается в игре отдельной модалкой. В нем есть две части:

- магазин материалов: покупка/продажа камня и железа;
- рынок игроков: покупка вещей, выставленных другими игроками.

Рынок игроков работает через Socket.io.

Игрок может выставить предмет на продажу. Сервер добавляет лот в `world.market`, сохраняет его в `server/world-state.json` и рассылает список всем клиентам.

При покупке лот удаляется с рынка, покупатель получает предмет в инвентарь.

Ограничение текущей версии: сервер не проверяет полноценно баланс покупателя и право собственности продавца, потому что авторизация пока локальная. Для production нужно переносить аккаунты, инвентарь и транзакции на серверную базу.

## Чат

Общий чат работает через Socket.io.

Функции:

- история последних сообщений;
- дата и время отправки;
- отправка под nickname;
- ответ на конкретное сообщение с цитатой;
- звуковой сигнал нового сообщения;
- включение/выключение звука чата с сохранением настройки в `localStorage`.

Сервер хранит историю чата в памяти процесса. После перезапуска сервера история очищается.

Новые товары на рынке и новые присоединившиеся игроки показываются игровыми уведомлениями.

## Сетевой Сервер

Файл: `server/multiplayer.mjs`.

Отвечает за:

- список онлайн-игроков;
- синхронизацию позиций игроков;
- состояние общей карты;
- строительство;
- золото;
- камень и железо;
- суперпризы;
- рынок;
- чат.

Основные события Socket.io:

```text
player:join

players:snapshot

player:leave

world:request

world:build
world:tile
world:gold:spawn

gold:collect

world:resource:spawn
resource:collect
resource:collected

prize:collect
world:prize:spawn
world:prize:despawn

market:list
market:sell
market:buy
market:bought

chat:history
chat:send
chat:message

online:snapshot
```

## Сохранения

Клиентские данные:

```text
lodegame-state-v1       профиль, прогресс, кастомизация
lodegame-auth-v1        локальные аккаунты
lodegame-chat-sound     настройка звука чата
```

Серверные данные:

```text
server/world-state.json
```

Не удаляйте `world-state.json`, если нужно сохранить построенный мир и рынок.

## Деплой

Next.js можно деплоить на Vercel, но Socket.io сервер и persistent `world-state.json` требуют отдельного Node.js-хостинга или отдельного сервиса.

Для production-версии нужно:

- вынести Socket.io сервер на постоянный Node.js сервер;
- заменить `world-state.json` на базу данных;
- сделать серверную авторизацию;
- проверять баланс, инвентарь и рыночные сделки на сервере;
- настроить `NEXT_PUBLIC_LODEGAME_SOCKET_URL` на адрес Socket.io сервера.

Пример переменной окружения:

```env
NEXT_PUBLIC_LODEGAME_SOCKET_URL=https://your-socket-server.example.com
```

## Текущие Ограничения

- Авторизация локальная, не защищенная сервером.
- Чат хранится в памяти Socket.io сервера.
- Рынок не является защищенной экономической системой.
- Persistent world хранится в JSON-файле, что подходит для прототипа, но не для большого онлайна.
- Multiplayer синхронизирует позиции и общие ресурсы, но не является полноценной authoritative-серверной симуляцией.

## Куда Развивать

- Перенести пользователей, профиль, инвентарь и рынок на серверную базу.
- Сделать authoritative-сервер для экономики и строительства.
- Добавить приватные сообщения, команды и модерацию чата.
- Добавить сохранение истории чата.
- Добавить редактор базы/территории и права доступа.
- Добавить полноценные серверные античиты для золота, предметов и скорости игрока.
